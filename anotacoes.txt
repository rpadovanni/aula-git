|------------------------------------------- LOGS --------------------------------------------|
	git log -p
		Mostra tudo que foi alterado em cada arquivo do commit.
		Exibe tudo dos logs.

	git log --stat
		Mostra estatisticas de alterações em cada commit. (2 insertions, 1 deletion).
		Exibe tudo dos logs.

	git log --pretty=oneline
		Exibe as mensagens de todos os commits em uma linha para cada commit.
		Exibe tudo dos logs.

	git log --pretty=format:"%h - %an, %ar : %s"
		Exibe informações de quem commitou, quanto tempo atrás e exibe a descrição.
		Exibe tudo dos logs.

	git log --since=2.days
		Exibe todos os commmits do periodo de tempo especificado.

|------------------------------------- VOLTANDO VERSÕES --------------------------------------|
	git checkout hash_do_commit
		Vai para versão especificada pelo Hash.
	
	git checkout master
		Volta para ultima versão commitada/branch principal.

	git reset HEAD~1
		Volta 1 commit.
		Pode se colocar o numero que quiser.

	git reset HEAD~1 --soft
		Volta X versões, porém deixa os arquivos das versões mais recentos prontos
		para serem comittados. Pode-se tirar do estágio 2, para editar o arquivo.

	git reset HEAD-1 --hard
		Volta X versões, apagando de vez os arquivos das versões mais recentes.

|---------------------------------------------------------------------------------------------|
|------------------------------------- CRIANDO BRANCHES --------------------------------------|
	git checkout -b nome_do_branch
		Cria um novo branch com o nome definido

	git checkou master
		Volta para branch principal

	git branch -a
		- Lista todos os branches, incluindo do repositorio online.


|---------------------------------------------------------------------------------------------|
|---------------------------------------- MERGE/REBASE ---------------------------------------|
	git merge nome_do_branch
		Faz o merge do branch citado no comando, para o branch selecionado.
		Cria um novo commit para juntar os commits de cada branch.

	git rebase nome_do_branch.
		Faz o merge do branch citado no comando, para o branch selecionado.
		Reorganiza os commits em ordem, para trazendo para o principal, para que não
		haja um terceiro commit.

|---------------------------------------------------------------------------------------------|
|------------------------------------------- CLONING -----------------------------------------|
	git clone endereço_git nome_do_diretorio_a_criar
		- Copia do github o repositório do link, criando a pasta com o nome passado
		como argumento.
		- Não é obrigatório o nome da pasta, só que ele cria com o nome do repositório
		online.

	git checkout -b nome_branch origin/nome_do_branch
		- Copia do github o branch especificado.

	git pull
		- Verifica se todos os arquivos foram baixados com sucesso do repositorio.

|---------------------------------------------------------------------------------------------|
|------------------------------------------ PUSH/PULL ----------------------------------------|
	- Para criar utilizar o repositorio em varios diretorios(computadores) diferentes
	é necessário sincronizar as alterações do repositorio com as do PC, com o comando
	"git pull".
